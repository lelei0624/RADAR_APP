How to Connect the
Marketing Intelligence Dashboard

Step-by-step guide to switching from demo mode to a live Databricks connection — covers credentials, CONFIG setup, the MKT Inbound filter, and how each view tab queries data.

⏱ Setup time: ~5 minutes
🗄️ Source: shared.revops
🔑 Filter: MKT Inbound
📊 Tables: 6 source tables · shared.revops
Setup Steps
1

2
Verify source tables
Confirm all 6 tables exist and have current data
The dashboard draws from six tables in shared.revops. Run these quick checks in the Databricks SQL Editor (warehouse: RevOps_SQL) before switching to live mode.
gold_monthly_exec_summaryPre-aggregated close_date KPIs — Won MRR, EmFi, CW Rev, Velocity, MQL/SQL counts. Rebuilt daily.
gold_targetsMonthly quota targets by segment and pipeline — MQL target, Won MRR target.
silver_dealsDeal-level HubSpot data — MQL/SQL funnel (create_date), won_mrr_cd, emfi_mrr_cd, channel source.
silver_line_itemsLine-item MRR per deal — joined with silver_deals to compute MQL TMRR and SQL TMRR by stage.
silver_companyCompany attributes — used for segment enrichment (SME Micro ratio, ENT profiling).
gold_metrics_datapediaMetric definitions and business glossary — reference table, not queried at runtime.
Quick health check — run in Databricks SQL Editor 
-- 1. Gold summary: check MKT Inbound rows are recent
SELECT MAX(metric_month) AS latest_month, COUNT(*) AS row_count
FROM   shared.revops.gold_monthly_exec_summary
WHERE  department_source_clean = 'MKT Inbound';

-- 2. Gold targets: confirm targets exist for this year
SELECT MAX(target_month) AS latest_target, COUNT(*) AS row_count
FROM   shared.revops.gold_targets
WHERE  department_source_clean = 'MKT Inbound';

-- 3. Silver deals: check create_date range and MKT Inbound volume
SELECT MAX(create_date) AS latest_create, COUNT(*) AS deal_count
FROM   shared.revops.silver_deals
WHERE  department_source_clean = 'MKT Inbound';

-- 4. Silver line items: confirm join key exists
SELECT COUNT(*) AS line_item_count
FROM   shared.revops.silver_line_items li
JOIN   shared.revops.silver_deals d ON li.deal_id = d.deal_id
WHERE  d.department_source_clean = 'MKT Inbound'
  AND  li.projected_close_date >= CURRENT_DATE();

-- 5. Silver company: spot-check
SELECT COUNT(*) AS company_count FROM shared.revops.silver_company LIMIT 1;
✅ What to check: Queries 1–4 should return non-zero row counts and a latest_month / latest_create within the last 2 days. If a table returns 0 rows, check that the upstream Databricks pipeline ran successfully under Jobs & Pipelines.

4
How live queries work
Understand what the dashboard queries
When demo: false, each view tab fires a query against gold_monthly_exec_summary filtered to MKT Inbound and the selected segment. The runDatabricksQuery() function handles the Databricks SQL REST API call.
Live API function (already in the file) 
async function runDatabricksQuery(sql) {
  const url = `https://${CONFIG.host}/api/2.0/sql/statements`;
  const resp = await fetch(url, {
    method: 'POST',
    headers: {
      'Authorization': `Bearer ${CONFIG.token}`,
      'Content-Type': 'application/json'
    },
    body: JSON.stringify({
      warehouse_id: CONFIG.httpPath.split('/').pop(),
      statement: sql,
      wait_timeout: '30s'
    })
  });
  const data = await resp.json();
  return data.result?.data_array || [];
}
To wire a view tab to live data, replace the DEMO[STATE.view][range] lookup inside render() with a call like this — one per view:
Example live query — Overall view, 30-day window 
const rows = await runDatabricksQuery(`
  SELECT
    metric_month,
    SUM(qualified_leads)         AS mql_count,
    SUM(sales_qualified_leads)   AS sql_count,
    SUM(won_deals)               AS won_deals,
    SUM(lost_deals)              AS lost_deals,
    SUM(won_mrr)                 AS won_mrr,
    SUM(won_ef_mrr)              AS emfi_mrr,
    SUM(won_amount)              AS cw_rev,
    SUM(mrr_in_pipeline)         AS tmrr,
    AVG(avg_deal_velocity)       AS velocity
  FROM ${CONFIG.schema}.gold_monthly_exec_summary
  WHERE department_source_clean = 'MKT Inbound'
    AND metric_month >= DATEADD(DAY, -30, CURRENT_DATE())
  GROUP BY metric_month
  ORDER BY metric_month ASC
`);
ℹ️ Date range filter: For the date range presets (7D / 14D / 30D / 90D), adjust the DATEADD(DAY, -N, CURRENT_DATE()) value. For custom ranges, pass STATE.customFrom and STATE.customTo as ISO date strings in the WHERE clause.
5
View Tabs
How each tab filters the data
Each tab applies a different filter on top of the base department_source_clean = 'MKT Inbound' condition. Add the appropriate AND clause to your live query per tab.
OverallNo segment filter — all MKT Inbound deals across all pipelines and segments.
SMEAND segment IN ('Micro', 'SME') — also shows Micro vs SME ratio breakdown.
ENTAND segment IN ('Mid-Market', 'Enterprise', 'ENT')
UpsellAND pipeline_name = 'PH Upsell Pipeline' AND department_source_clean = 'MKT Inbound' — replaces MQL with UQL; shows per-product TMRR breakdown.
AboutStatic documentation tab — no query.
✅ MKT Inbound filter: All views apply department_source_clean = 'MKT Inbound' as the base filter. This field maps to the original source of the deal in HubSpot. Renewals are excluded via LOWER(deal_name) NOT LIKE '%renewal%' in the upstream silver_deals table.
6
TMRR Stages
Understanding MQL TMRR vs SQL TMRR
Both TMRR metrics come from mrr_in_pipeline in the gold table, which sums line item MRR for deals in active stages with a future projected_close_date. Filter by client_journey_stage to split them:
MQL TMRR — includes Appointment Scheduled stage 
-- MQL TMRR: Appointment Scheduled + all active pipeline stages
SELECT SUM(mrr_in_pipeline) AS mql_tmrr
FROM   shared.revops.gold_monthly_exec_summary
WHERE  department_source_clean = 'MKT Inbound'
  AND  stage_label IN (
         'Appointment Scheduled',
         'Initial Demo Done',
         'Solutioning',
         'Commercial Negotiation',
         'Committed Accounts'
       );
SQL TMRR — later-stage pipeline only 
-- SQL TMRR: deals that have passed initial qualification
SELECT SUM(mrr_in_pipeline) AS sql_tmrr
FROM   shared.revops.gold_monthly_exec_summary
WHERE  department_source_clean = 'MKT Inbound'
  AND  stage_label IN (
         'Solutioning',
         'Commercial Negotiation',
         'Committed Accounts'
       );

Troubleshooting
❗ Error: "401 Unauthorized"
Your Personal Access Token is wrong or expired. Go back to Databricks → Settings → Developer → Access Tokens, generate a new one, and replace the token value in CONFIG.
❗ Error: "Failed to fetch" / CORS error
You're opening the file directly from your filesystem (file:// URL). Use a local HTTP server instead — run "python -m http.server 3000" in the MKTNG DB folder and open http://localhost:3000/sample_dashboard.html in Chrome.
❗ Charts load but all values are 0 or empty
The query returned rows but no MKT Inbound data for the selected date range. Check that:
1. department_source_clean = 'MKT Inbound' exists in the gold table (run Step 2 query)
2. The gold table was rebuilt recently (check last pipeline run in Databricks Jobs)
3. The selected date range (7D / 30D / custom) overlaps with the available data
❗ "Table not found" error
The gold_monthly_exec_summary table hasn't been created yet in shared.revops. Run the CREATE OR REPLACE TABLE script manually in the Databricks SQL Editor to build it from silver_deals and silver_line_items.
